# PRICE GUARD — EXECUTION PLAN

## Phase 1: Public Free MVP (Website Tool)

---

## 1. Product Positioning

**Price Guard**
A deterministic pricing calculator for makers and small sellers.

Core framing:

> Prevent selling at a loss.

This is not “a calculator.”
It is a **pricing guardrail engine**.

---

## 2. Current Technical Baseline (From Repo + Context)

### Stack

* TypeScript (84%)
* Local-only browser execution
* Static site deployable to Cloudflare Pages
* SoT-first workflow enforced
* Unit tests included
* Deterministic pricing engine
* Rounding modes present
* Solver iterations already tightened

### Pricing Engine Structure (from earlier interface discussion)

Inputs:

* Materials
* Packaging
* Labor (minutes + rate)
* Overhead (percent or fixed)
* Platform fee (percent + fixed)
* Shipping (optional inclusion)
* Pricing goal (margin or profit)
* Rounding mode

Outputs:

* Break-even
* Recommended price
* Profit
* Margin
* Cost breakdown
* Impossible-state detection (if implemented)

Math philosophy:

* Integer cents internally
* Rounding applied post-calculation
* Deterministic solver logic

This foundation is solid.

---

## 3. MVP Scope (Free Website Version)

The first public version must:

### Must Have

* Required price calculation
* Reverse calculation mode (sale price → actual profit)
* Cost breakdown display
* Platform fee transparency
* Margin safety indicator
* Psychological rounding options
* Offline-capable PWA
* Clean branding footer

### Must Not Have (Yet)

* PDF export
* Save profiles
* Multi-scenario compare
* Custom logo upload
* Account system
* Cloud sync

Free version = Trust engine.

---

## 4. UX Improvements Before Public Launch

### A. Safety Indicator System

Add dynamic state messaging:

* Loss → Red warning
* Margin < 10% → Amber caution
* Margin target achieved → Green confirmation

Positioned above results.

Purpose: Emotional reinforcement.

---

### B. Platform Take Emphasis

Add explicit:

“Platform keeps: $X (Y%)”

This is powerful psychologically and shareable.

---

### C. Reverse Mode

Toggle:

* Calculate Required Price
* Calculate Profit From Sale Price

This is high-value and differentiating.

---

### D. Rounding Expansion

Add:

* .99
* .95
* Ceiling dollar
* Floor dollar

Applied after full math resolution.

Unit test rounding edge cases.

# **BUS Core — Landing Page Redesign**

## **Product Category**

BUS Core is best positioned as:

**Operational infrastructure for small manufacturing shops.**

Not:

* enterprise ERP  
* SaaS business software  
* hobby maker tool

Instead it sits in the gap between spreadsheets and ERP.

```
Spreadsheets  →  BUS Core  →  ERP
```

This segment represents the **“messy middle”** of small production operations.

Typical users:

* laser engraving shops  
* 3D print farms  
* small fabrication shops  
* Etsy-scale manufacturing  
* garage product makers

These operators typically manage production through:

* spreadsheets  
* notebooks  
* memory

BUS Core replaces this with a structured workflow system.

---

# **Core Positioning**

## **Primary Tagline**

**BUS Core — a digital shop jig for manufacturing workflows**

## **Supporting Statement**

Track materials, production runs, and real costs so small manufacturing shops can build consistently without spreadsheets.

---

# **Why the “Shop Jig” Metaphor Works**

In manufacturing, a jig exists to ensure repeatable production.

It:

* constrains motion  
* enforces process  
* guarantees consistency

BUS Core applies this concept to operational workflows.

It constrains and structures the process of:

* tracking materials  
* running production batches  
* recording inventory movement  
* calculating product cost  
* maintaining a production ledger

This metaphor resonates strongly with fabrication and maker audiences because it reflects real shop practices.

---

# **Audience Psychology**

The intended users typically share several traits:

### **Skeptical of SaaS**

Many small manufacturing operators prefer tools that run locally and retain data ownership.

### **Tool-Oriented Thinking**

They respond well to systems that resemble tools or infrastructure rather than abstract business platforms.

### **Operational Focus**

They care about:

* stock levels  
* production cost  
* batch history  
* material consumption

They care less about:

* dashboards  
* productivity buzzwords  
* corporate language

### **Technical Comfort**

Many small shops already operate equipment that requires technical understanding:

* CNC machines  
* laser cutters  
* slicer software  
* fabrication tools

They are comfortable with technical interfaces if the tool is practical.

# **Appendix — Proven Infrastructure Tool Homepage Layout**

Many successful technical infrastructure products follow a very similar homepage structure.  
Examples include Docker, Fly.io, Supabase, Tailscale, and GitHub.

These sites are designed for **technical operators rather than marketing buyers**, and their layouts prioritize clarity, capability, and installation speed.

BUS Core fits naturally into this design pattern.

---

# **Infrastructure Homepage Layout Model**

The structure typically follows this sequence:

```
Hero
↓
Recognition (Problem)
↓
System Concept
↓
Core Capabilities
↓
Architecture / Credibility
↓
Screenshots / Demonstration
↓
Installation / Getting Started
↓
Philosophy / Community
```

Each section has a specific purpose in guiding technical users toward adoption.

* brief supporting description  
* primary call-to-action (Install / Get Started)
